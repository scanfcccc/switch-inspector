## 勘探发现

### 外部研究
- **pluggy** (pytest 内核): hookspec/hookimpl 模式，LIFO 执行
- **entry_points**: pip 包插件标准发现
- **`__init_subclass__`**: 同项目内零配置注册
- **stevedore**: 生产级插件管理

### 关键发现
- **零测试**: 没有任何测试文件、框架
- **全局状态**: registry/catalog 在 FastAPI lifespan 中初始化
- **报告逻辑全部硬编码**: build_report() 中的阈值、检查区间都是魔法数字

## 确认的决定
- **目标用户**: 混合 — 简单插件用声明式配置（YAML），复杂插件用 Python API
- **插件化范围**: 告警规则、合规检查、字段处理器、文件适配器
  - 明确不包含: 导出器、预览器（当前不在此方案范围）
- **分发方式**: 项目内目录 + pip entry_points 双通道
- **容错策略**: 分层 — 验证后加载，运行时出错则隔离跳过
- **配置管理**: 混合 — 插件自声明 schema，用户按 schema 在项目 YAML 提供值
- **动态加载**: 分环境 — 开发模式热重载（watchdog），生产模式重启加载
- **插件依赖**: 管道链 — 字段处理器可形成处理管道（上游出 → 下游入）
- **实施策略**: 先修补 → 再拓展 — 补足现有 parsers 插件不足，再拓展新插件类型
- **兼容策略**: 渐进迁移 — 旧接口标记 deprecated，留过渡期
- **测试策略**: TDD — 新插件接口必须有测试
- **MVP 范围**: 单类插件 + 规范 — 选告警规则为试点，完整重构为插件 + 输出规范文档
